## Abstract

This working paper records a bounded Project Telos pass over biological
network-intelligence literature. The goal is not to turn fungal, mycorrhizal,
or plant signaling into a cognition claim. The goal is to extract a disciplined
engineering hypothesis for large-context AI systems: move sparse signals
through a durable substrate, retrieve high-dimensional evidence by receipt only
when needed, reinforce useful routes, and decay routes that produce drift.

The current evidence rung is a Gather corpus with ten stored, distinct,
source-bound items and a verified run digest. The corpus includes fungal
electrical signaling, plant glutamate/calcium/bioelectric signaling, common
fungal or mycorrhizal network evidence, and critique/response material around
overinterpreted network narratives. It supports the hyphal context protocol as
an architectural hypothesis. It does not prove that hypothesis as a deployed
system or benchmark result.

## What The Corpus Actually Supports

The verified corpus supports source-bound statements about literature coverage:

- fungal electrical signaling and spike-like responses are active research
  topics;
- plant glutamate receptor, calcium, and bioelectric signaling mechanisms are
  relevant to distributed biological information flow;
- common fungal and mycorrhizal networks have specific experimental and review
  coverage;
- broad network narratives are contested and require careful claim boundaries;
- source accessibility must be represented explicitly when DOI routes are
  blocked.

The engineering inference is narrower than the biology:

> A durable substrate can carry state, while low-dimensional signals route
> attention toward evidence receipts that can be rehydrated on demand.

That inference is useful for Telos, but it remains a hypothesis until measured.

## Hyphal Context Protocol

The proposed Telos primitive is a hyphal context protocol:

1. Store full evidence bodies in Gather and Index rather than in every model
   prompt.
2. Emit sparse gradient envelopes when evidence, route pressure, contradiction,
   budget pressure, or operator preference changes.
3. Route those envelopes through Forum using local path scores and prior
   receipt outcomes.
4. Retrieve high-dimensional evidence only when a downstream task requires it.
5. Ask Crucible to classify claims as `MATCH`, `DRIFT`, or `UNVERIFIABLE`.
6. Convert verified packets into Learn prooflessons so operators can study the
   boundary without converting the lesson into unsupported claims.

This is not a claim that biology has already solved agent orchestration. It is
a design thesis for receipt-first AI infrastructure.

## Product Implications

Gather should add source availability fields such as `open`, `abstract-only`,
`publisher-blocked`, `author-copy`, and `unverifiable-from-local-corpus`.

Index should add gradient-envelope records for changed claims, missing links,
freshness roots, downstream consumers, and evidence pressure.

Forum should route agents and tools through local path reinforcement, not only
keyword matching or a single global planner.

Crucible should add negative tests for biological overreach and positive tests
for evidence retention under token pressure.

Learn should generate lessons that teach why a bounded source packet licenses
architecture hypotheses but not broad biology claims.

BuildLang/buildc should become the runtime for reproducible benchmark receipts:
the same routing problem, the same evidence corpus, two execution strategies,
and machine-readable cost/output/verdict records.

## Benchmark Target

The first benchmark should compare two routes for one research task:

| Route | Input | Expected advantage | Failure mode |
| --- | --- | --- | --- |
| Full-context route | all candidate source summaries | fewer retrieval misses | high token cost and more unsupported synthesis |
| Hyphal route | gradient envelopes plus receipt IDs | lower token cost and stronger provenance joins | missing evidence if retrieval triggers are weak |

Minimum measurements:

- tokens spent,
- relevant evidence recovered,
- unsupported claims emitted,
- Crucible `MATCH` / `DRIFT` / `UNVERIFIABLE` verdicts,
- ability to resume after compaction or context loss,
- and whether a later agent can join every claim to a receipt.

## Source Boundary

This packet used only accessible web, PubMed/PMC, and arXiv routes in the local
Gather store. Two DOI routes were retained as leads after HTTP 403 responses.
That is a feature of the packet discipline: inaccessible sources should become
explicit coverage gaps, not silent citations.

## Tooling Thesis

This pass tests the broader Telos biology-to-infrastructure pattern:

1. Gather records biological source leads with hashes and refs.
2. The source gate records blocked DOI routes as explicit coverage gaps.
3. Crucible receives a bounded thesis and measurements.
4. Learn converts the packet into a prooflesson so the operator can study the
   claim boundary without converting the lesson into an answer dump.
5. BuildLang/buildc remains the next runtime target for deterministic
   benchmark receipts.

Crucible assessed three bounded claims as `MATCH`: the corpus contains ten
stored, distinct items and verifies; the publication copy keeps biological
inspiration separate from overreach; and the hyphal context protocol is stated
as a hypothesis requiring benchmark evidence.
