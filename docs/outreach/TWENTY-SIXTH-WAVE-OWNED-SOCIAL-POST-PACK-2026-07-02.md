## LinkedIn Long Posts

### L01 - Receipts, not trust

Exact text:

```text
AI agents do not need more trust language.

They need receipts.

The failure mode I keep seeing is simple: an agent does useful work, but nobody can answer the hard questions later.

What context did it see?
Which tools were available?
What did it call?
What changed?
What evidence supports the final claim?
Which assumptions were left unresolved?

Project Telos is my attempt to make that record native.

The current public stack ties source intake, workspace indexing, agent routing, verification pressure, and action receipts into a local MCP/CLI surface. The point is not "another agent framework." The point is making AI-assisted work inspectable after the moment has passed.

If a model answer matters, it should leave a trail a reviewer can re-check tomorrow.

Repo: https://github.com/HarperZ9/telos
```

### L02 - MCP vs CLI

Exact text:

```text
MCP vs CLI is the wrong fight.

CLI is excellent when the operation is mature, composable, and already compact.

MCP is useful when a host needs typed tool metadata, dynamic capability discovery, and a standard bridge across clients.

The real problem is tool-surface selection.

If an agent sees 90 tools, the interface has already failed. If it sees only the few tools relevant to the current phase, and the host records the exact schemas and descriptions it saw, MCP becomes much more useful.

That is the direction Project Telos is taking: context envelopes, narrow tool exposure, action receipts, and verifier gates.

The interface matters less than whether the operator can prove what happened.

Repo: https://github.com/HarperZ9/telos
```

### L03 - Model foundry

Exact text:

```text
Project Telos is not trying to pretend a workstation can reproduce frontier-lab pretraining.

The model direction is more practical:

- hosted frontier APIs when policy and privacy allow
- local/open-weight models for private, cheap, or offline work
- post-training experiments where the size is feasible
- tool-use surfaces through CLI and MCP
- context envelopes for large workspaces
- eval and safety gates before promotion

I think the interesting "model foundry" layer is around models, not only inside them.

Routing, receipts, eval deltas, source refs, and promotion gates are where a lot of real-world value sits.

Repo: https://github.com/HarperZ9/telos
```
