Purpose: capture the current updated tool surface after the recent tool refresh, separate installed command evidence from local repo evidence, and identify what is ready to package for demos and outreach.

## Executive Shape

Project Telos is now best described as a proof-native AI workbench and model-building operating layer. The center is not one app. The center is a set of re-checkable membranes:

- Gather turns source intake into receipts.
- Index turns workspace structure into bounded context envelopes.
- Forum turns intent into routed lanes and reusable context capsules.
- Crucible turns claims and measurements into `MATCH`, `DRIFT`, or `UNVERIFIABLE`.
- Telos joins the flagships into an MCP/catalog/workflow surface with action receipts, context packs, doctor lanes, model-foundry planning, learning labs, browser evidence, and demo packaging.
- Learn turns study and proof packets into receipt-backed learning loops.
- BuildLang/buildc turns language/runtime behavior into effect, policy, compiler, corpus, and backend receipts.
- Emet is the external source/view witness for byte-level and view-level consistency.

The current practical wedge is stronger than the earlier research-only framing: Telos can now show live host compatibility, tool-surface freshness, source federation, context selection, ledger capsule handoff, and operator-doctor verification as one product loop.
